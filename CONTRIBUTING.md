# Contributing to WAR

Thanks for your interest in contributing.

## Ground Rules

- Be respectful and constructive.
- Keep pull requests focused and small.
- Open an issue before large changes.

## Development Workflow

1. Fork and clone the repository.
2. Create a feature branch: `feature/short-description`.
3. Make your changes with tests where possible.
4. Run lint and tests before pushing.
5. Open a pull request with a clear summary.

## Pull Request Checklist

- [ ] Code builds successfully.
- [ ] Tests were added or updated for behavior changes.
- [ ] Documentation was updated when needed.
- [ ] No unrelated files were changed.

## Commit Message Guideline

Prefer clear, scoped messages, for example:

- `feat(engine): add turn phase transition`
- `fix(api): validate card target slot`
- `docs(readme): update setup steps`

## Reporting Bugs

Please include:

- Expected behavior
- Actual behavior
- Reproduction steps
- Environment details (OS, Node version)

## Feature Requests

Please include:

- Problem statement
- Proposed solution
- Alternatives considered
