# Security Policy

## Supported Versions

Until versioned releases are established, only the latest `main` branch is considered supported.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use one of these options:

1. Open a private vulnerability report if the hosting platform supports it.
2. Contact the maintainers directly with details.

Include:

- Description of the issue
- Impact assessment
- Reproduction steps or proof of concept
- Suggested mitigation if available

## Response Targets

- Initial response: within 7 days
- Triage and confirmation: within 14 days when reproducible
- Fix timeline: depends on severity and complexity
